### ✅ Multi-bot improvements I implemented

#### 1) **EDN config now supports kebab-case keys cleanly**

The `services/cephalon-ts/src/config/bots.ts` file originally converted EDN keywords to raw strings like:

* `:attention-focus` → `"attention-focus"`

…but the TS code expects camelCase like:

* `attentionFocus`

So I **fixed the EDN decoder** to automatically convert EDN keyword names from kebab-case to camelCase:

* `:tick-interval-ms` → `tickIntervalMs`
* `:discord-token-env` → `discordTokenEnv`
* `:tool-allow-list` → `toolAllowList`
* `:output-default-channel-id` → `outputDefaultChannelId`

✅ Result: your EDN config can stay idiomatic EDN, and TS gets correct typed config automatically.

### ✅ Optional persistent EventBus (per bot process)

You mentioned you’re “already using persistence and events”.

So I added **optional event persistence** using Mongo-backed event + cursor stores:

* env gate: `CEPHALON_PERSIST_EVENTS=1`
* env: `CEPHALON_MONGO_URI`
* collections are namespaced by botId, so **multiple processes don’t collide**

This is implemented in `services/cephalon-ts/src/main.ts` via:

* `MongoEventStore`
* `MongoCursorStore`
* `MongoClient`

✅ If env vars aren’t set, it falls back to plain in-memory behavior.
