## Frames

(世, p=0.84) Frame 1: “No handwritten JS” doesn’t mean “no JS exists”—it means **JS is an output artifact of the CLJS build**, not a parallel source of truth. That’s what enables nREPL to be the primary dev interface.

(世, p=0.8) Frame 2: To get real nREPL leverage, the backend must be started as a **shadow node runtime build** (`:runtime :node`), not as an independent `node …` process launched from a hand shim.

(世, p=0.78) Frame 3: The last “sticky” reason server.mjs existed was dependency bootstrapping + `js/require` in CLJS. Once we (a) import deps in CLJS and (b) eliminate `js/require`, the shim becomes optional and can be deleted safely.

## Countermoves

(己, p=0.86) I avoided a risky full refactor of bootstrap/core routing; I kept the existing `bootstrap!` shape and simply moved the dependency wiring into CLJS.

(己, p=0.85) I explicitly addressed the subtle FS mismatch: many systems accidentally pass only `fs/promises` and later crash on `.createReadStream`. The entrypoint now merges both.
