 Signal
 It is effectively “if policy doesn’t fully decide, here is the ambient list of
 providers to try.” That is exactly wrong for the architecture you’re describing.

 Evidence

 services/proxx/policies/runtime/05-provider-seed.edn is already the policy-owned
 provider seed surface.

 Right now it declares:

 - provider identity env names
 - provider ID fallback
 - API key env names

 But base URLs / routable provider endpoints still come from:

 - UPSTREAM_PROVIDER_BASE_URLS
 - hardcoded defaults in src/lib/config.ts
 - route-building code that enumerates that ambient map

 So yes: those “upstream providers” are not neutral config. They are a second
 routing authority.

 Frames

 The correct model should be:

 1. Policy declares provider existence.
 2. Policy declares provider endpoint/base URL.
 3. Policy declares model/provider eligibility.
 4. Policy declares strategy.
 5. Policy declares account selection.
 6. Runtime executes the one policy decision.

 The wrong current model is:

 1. Env/config declares an ambient provider map.
 2. TS builds candidates from it.
 3. CLJS policy narrows/reorders those candidates.
 4. TS still retains escape hatches/fallback behavior around that narrowed result.

 That means policy is not sovereign yet; it’s advisory-plus-filtering over a legacy
 provider universe.

 Countermoves

 The next right move is not to tune UPSTREAM_PROVIDER_BASE_URLS.

 The next right move is to make 05-provider-seed.edn / policy EDN carry provider
 route facts, e.g. conceptually:

 ```edn
   {:contract/id :provider-seed/llamacpp
    :contract/kind :provider-seed
    :provider-id-fallback "llamacpp"
    :provider/base-url "http://127.0.0.1:8082"
    :key-env-names ["LLAMACPP_API_KEY"]}
 ```

 or a sibling policy contract:

 ```edn
   {:contract/id :provider-route/llamacpp
    :contract/kind :provider-route
    :provider/id "llamacpp"
    :provider/base-url "http://127.0.0.1:8082"}
 ```

 Then CLJS policy preview can return concrete :provider-routes, and the legacy env
 map becomes unnecessary.

 Next

 to kill.
