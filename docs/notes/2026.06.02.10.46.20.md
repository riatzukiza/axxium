## Axxiums

- Users can create accounts with any or all of  the following at first:
  - discord
  - github
- When a node is created, an SSH/RSA public/private key pair is created
- When a node registers with a relay, an SSH tunnel is created with the public key
- relays never have private keys
- the policy engine can make  decisions based on any fact  that can be durived  from the associated sources  of authentication such as:
  - discord user name
  - membership in a discord server
  - A role assigned to a user with in a discord server
  - git org membership
  - git org role/permissions
  - anything you can durive from any of these API
- The interface allows users to create Bearer tokens to use to access the API instead of their oauth for easy access after the account is initially created
- peer nodes must always be associated with a user
- policy decisions made on actions taken by peer nodes can only make decisions deducable from the peers user DID
  - policy does not know a users IP address


## Initial relay policies
- gemma4:31b is available to all ussyverse discord  server members with generous average request per 10 minute limits
- Ussyverse server members with the Trust role have unlimited gemma4:31b access
- Trust members have access to all models with per 10 minute rate limits when available
- Trust members may connect to the relay as peer nodes

## Nodes
- store credentials
- governed by local policy
- are owned by a user
- axxiumaticly allow all actions to be taken  by their owner
- Axxiumaticly disallow  by default all actions not described by policy to any other user
